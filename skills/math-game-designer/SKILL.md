# 小学数学教学游戏设计师

## 概述

本技能将详细的数学教学步骤转化为创意十足、可玩的HTML游戏原型。
针对每个知识点，设计5个独立的游戏方案——每个方案采用不同的游戏机制，
培养不同的思维模式——然后构建功能完整的HTML原型，让学生
可以直接在浏览器中互动体验。

输出内容为5份游戏设计文档（GDD）+ 5个可玩的HTML原型。每个原型
内置自适应练习系统——游戏结束后，学生自动获得
根据游戏表现匹配难度的练习题。

## 工作流程

### 第一步：分析教学步骤

当用户提供教学步骤时，首先进行系统分析：

1. **识别知识点** — 教的是什么数学概念？（如"两位数加法进位"、"分数的基本性质"）
2. **确定年级** — 根据教学步骤的复杂度和用语推断。参考 `references/grade_level_guidelines.md` 了解各年级认知特征。
3. **提取学习目标** — 学完后学生应该能做什么？
4. **识别常见误区** — 学生通常在哪些地方遇到困难？
5. **记录前置知识** — 学生需要已掌握哪些知识？

在进入游戏设计前，以简洁的摘要块输出此分析。

### 第二步：生成5份游戏设计文档

设计5款游戏，要求：
- **创意且各异** — 每款使用根本不同的游戏机制，绝不重复同类机制。
- **启发思考** — 游戏需要推理，而非机械练习。尽量包含开放式挑战。
- **适龄** — 按 `references/grade_level_guidelines.md` 匹配认知水平。
- **教学合理** — 每款游戏必须与输入中的具体教学步骤有明确关联。

选择游戏机制时，参考 `references/game_mechanics_catalog.md`（12个族系、82种机制，含知识点匹配矩阵）。

**机制选择流程：**
1. 首先查阅目录底部的"知识点 × 玩法速查矩阵"——找出标记为 ✅（高度适配）或 ○（可以适配）的机制。
2. 从候选中选出5个机制，跨越**5个不同族系**。同一族系不能出现两款游戏。
3. 优先选择适用年级范围覆盖目标年级的机制。
4. **交叉参考 `references/international_game_library.md`**：在游戏库中查找目标知识点的分类，了解现有网页游戏如何处理类似数学概念。研究其交互模式、反馈系统和难度递进，为HTML原型设计提供灵感。

**族系覆盖要求：**
5款游戏必须来自5个不同族系。可接受的组合涵盖12个族系中的任意5个：

| 编号 | 族系 | 关键词 | 适合转化HTML的程度 |
|:----:|------|--------|:-----------------:|
| A | 竞速族 | 快、先、抢、限时 | ★★★★★ 天然适配 |
| B | 卡牌族 | 抽、翻、配、组 | ★★★★★ 天然适配 |
| C | 棋盘族 | 掷骰、走格、路径 | ★★★★ 棋盘需模拟 |
| D | 操作族 | 拼、摆、量、画、折 | ★★★ 需模拟操作界面 |
| E | 推理族 | 猜、排、破、推 | ★★★★★ 天然适配 |
| F | 角色族 | 扮、模拟、交易 | ★★★★ 情景模拟 |
| G | 对抗族 | 攻、防、比、赌 | ★★★★ 需要AI对手 |
| H | 合作族 | 协作、共建、教 | ★★★ 单人多角色模拟 |
| I | 运气族 | 随机、抽奖、悬念 | ★★★★★ 天然适配 |
| J | 韵律族 | 歌谣、拍手、节奏 | ★★ 身体参与难转化 |
| K | 笔纸族 | 连线、填空、涂色 | ★★★★★ 天然适配 |
| L | 运动族 | 跑、跳、投、蹲 | ★ 身体运动难转化 |

对于HTML原型，优先选择 ★★★★ 及以上的族系。J/L族系可以做概念性改编，但需要为屏幕交互做大幅重新设计。

每款游戏按 `references/game_design_doc_template.md` 模板生成游戏设计文档，GDD须包含：

1. **游戏名称** — 朗朗上口、中文、适合年龄段
2. **游戏机制类型** — 来自机制目录，含目录编号（如"A01 抢答赛"、"E02 密室逃脱"）
3. **核心概念** — 一段话的简要描述
4. **学习目标映射** — 本游戏对应哪些教学步骤？
5. **核心游戏循环** — 玩家操作的步骤描述
6. **思维模式** — 激发什么思维？（逻辑推理/空间想象/策略规划/创造性思维/归纳总结/...）
7. **视觉风格** — 简要美术方向说明
8. **难度递进** — 游戏如何提升难度？
9. **反馈系统** — 游戏如何告知学生对错？
10. **HTML原型范围** — 原型包含哪些功能？

### 第三步：构建含自适应练习的HTML原型

为每款游戏构建独立的、可玩的HTML文件。每个原型必须包含两个集成
系统：**核心游戏**和**自适应练习模块**。

**技术要求：**
- 单个 `.html` 文件——无外部依赖（内联所有CSS/JS）
- 可离线运行——无CDN、无网络请求
- 响应式——适配桌面浏览器和平板（最小宽度：320px）
- 触控友好——按钮/可点击区域至少44×44px
- 中文界面
- 简洁、适合儿童的视觉设计

**游戏性要求：**
- 明确的胜负条件
- 每次操作即时反馈
- 分数或进度追踪
- "重新开始"按钮
- 至少3个关卡/回合
- 游戏内递进难度

**自适应练习要求（嵌入游戏内）：**
- 游戏结束后，胜利画面显示"进入练习"按钮
- 练习难度根据游戏正确率**自动选择**：
  - 正确率 < 40% → 第1档（基础巩固）
  - 正确率 40-60% → 第2档（查漏补缺）
  - 正确率 60-80% → 第3档（能力提升）
  - 正确率 80-95% → 第4档（思维拓展）
  - 正确率 >= 95% → 第5档（挑战冲刺）
- 每档包含3-5道题，来自按 `references/practice_problems_template.md` 定义的5档题库
- 练习模式即时显示对错反馈及解题提示
- 练习结束后显示汇总画面，合并游戏+练习成绩
- 始终提供"直接重玩"跳过选项，可跳过练习

**代码质量：**
- 注释良好的JavaScript
- 游戏逻辑、练习逻辑和渲染层分离清晰
- 游戏过程中追踪 `correctCount` 和 `totalAttempts` 用于档位计算
- 处理边界情况（快速连点、浏览器缩放、空输入等）
- 包含带游戏名称的 `<title>` 标签

以 `assets/game_template.html` 作为起始脚手架。模板已包含：
- 完整练习模式UI（默认隐藏）
- `determinePracticeTier()` 自适应选择逻辑
- `enterPracticeMode()` / `renderPracticeQuestion()` / `showPracticeSummary()` 流程
- 选择题和填空题类型及反馈

**各游戏定制方法：**
1. 将游戏逻辑部分替换为你游戏的机制
2. 将 `practiceProblems` 对象替换为5档真实题目
3. 按需自定义 `difficultyLabels` 档位名称
4. 确保游戏逻辑追踪 `state.correctCount` 和 `state.totalAttempts`

### 第四步：生成练习题池

为当前知识点设计5档练习题池，将嵌入每个游戏原型中。
遵循 `references/practice_problems_template.md`。

**关键设计规则：**
1. 5档中每档3-5道适当难度的题目
2. 第1-2档：直接应用，建立信心。第3档：多步骤，加深理解。
   第4档：新情境，迁移学习。第5档：开放式，创造性思维。
3. 题型混合：题库中至少一道填空题和一道开放/推理题
4. 每道题必须包含 `hint`——简要的解题思路，重在教会方法，而非直接给答案
5. 同一题库在5个游戏原型中复用（每个游戏变化的是玩法，
   而非练习内容）

**写入每个游戏原型：**
将 `practiceProblems` 对象复制到每个HTML文件的 `<script>` 部分，替换
模板中的示例数据。练习UI和自适应逻辑已内置于模板中。

## 游戏设计原则

每款游戏设计都遵循以下原则：

### 1. 先思考，再操作
游戏应需要动脑。避免学生靠随机点击就能通关的设计。每次交互都应是一个决策。

### 2. 错误即学习
错误答案应有教育意义，而非惩罚。展示**为什么**答错了。允许重试。绝不使用负面语言或羞辱。

### 3. 让数学可见
让数学思维可视化。使用数轴、面积模型、教具和视觉表征。不要把数学藏在花哨效果后面。

### 4. 递进挑战
先从直接应用开始建立信心，再引入需要更深层理解的变化。最后一关应让学生将知识迁移到新情境。

### 5. 赞赏思考，不只是答案
奖励策略、进步和创造性方法——不仅仅是正确答案。包含"思考加分"或对巧妙解法的认可。

### 6. 自适应练习形成闭环
游戏是吸引点——练习才是学习。内置的自适应练习系统确保每个学生都能获得适合自己水平的题目：薄弱学生巩固基础，优秀学生挑战更高。这避免了静态练习"一刀切"的困境。

### 7. 中国数学教育情境
- 对齐人教版课程标准
- 使用中国数字格式和惯例
- 如涉及，遵循中国股市色彩惯例（红涨绿跌）
- 支持所有数字形式的中文字符显示
