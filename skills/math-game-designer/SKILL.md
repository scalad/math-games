# 小学数学教学游戏设计师

## 概述

本技能将详细的数学教学步骤转化为富有创意、可交互的 HTML 游戏原型。
针对每个知识点，设计5个独立的游戏方案——每个方案使用不同的游戏机制，
侧重不同的思维模式——然后构建功能完整的 HTML 游戏原型，学生可直接在浏览器中操作。

最终产出为 5 份游戏设计文档（GDD）+ 5 个可玩的 HTML 游戏原型。每个原型
都内置了自适应练习系统——游戏通关后，学生根据游戏中的表现自动获得匹配其能力水平的练习题。

## 工作流程

### 步骤一：分析教学步骤

当用户提供教学步骤时，首先系统性地进行分析：

1. **确定知识点** —— 正在教授什么数学概念？（如"两位数加法进位"、"分数的基本性质"）
2. **确定年级** —— 根据教学步骤的复杂度和语言推断年级。参考 `references/grade_level_guidelines.md` 了解各年级认知特征。
3. **提取核心教学目标** —— 学习结束后学生应该能够做什么？
4. **识别常见误区** —— 学生通常在哪里遇到困难？
5. **标注前置知识** —— 学生必须已经掌握哪些内容？

在进入游戏设计之前，先输出一个简洁的分析摘要。

### 步骤二：生成 5 份游戏设计文档

设计的 5 款游戏应满足：
- **富有创意且各不相同** —— 每款游戏使用根本不同的游戏机制。绝不重复同一机制类型。
- **激发思考** —— 游戏应要求推理，而非机械练习。尽可能包含开放性挑战。
- **适龄** —— 根据 `references/grade_level_guidelines.md` 匹配认知水平。
- **教学合理** —— 每款游戏必须清晰对应输入材料中的具体教学步骤。

选择游戏机制时，查阅 `references/game_mechanics_catalog.md`（12大族系，82种玩法，含知识点匹配矩阵）。

**机制选择工作流：**
1. 首先查阅总表底部的"知识点 × 玩法速查矩阵"——找到目标知识点对应标记为 ✅（高度适配）或 ○（可以适配）的机制。
2. 从候选机制中，挑选覆盖 **5个不同族系** 的 5 种机制。同一个族系不能有两款游戏。
3. 优先选择适用年级范围覆盖目标年级的机制。
4. **与 `references/game_mechanics_catalog.md` 交叉参考**：该文件已合并国际游戏参考库（第二部分），在游戏库中查找目标知识点对应的分类，了解现有网页游戏如何处理类似的数学概念。研究它们的交互模式、反馈系统和难度递进，为 HTML 原型设计获取灵感。

**族系覆盖要求：**
5 款游戏必须来自 5 个不同的族系。可接受的组合涵盖以下 12 个族系中的任意 5 个：

| 编号 | 族系 | 关键词 | 适合转化HTML的程度 |
|:----:|------|--------|:-----------------:|
| A | 竞速族 | 快、先、抢、限时 | ★★★★★ 天然适配 |
| B | 卡牌族 | 抽、翻、配、组 | ★★★★★ 天然适配 |
| C | 棋盘族 | 掷骰、走格、路径 | ★★★★ 棋盘需模拟 |
| D | 操作族 | 拼、摆、量、画、折 | ★★★ 需模拟操作界面 |
| E | 推理族 | 猜、排、破、推 | ★★★★★ 天然适配 |
| F | 角色族 | 扮、模拟、交易 | ★★★★ 情景模拟 |
| G | 对抗族 | 攻、防、比、赌 | ★★★★ 需要AI对手 |
| H | 合作族 | 协作、共建、教 | ★★★ 单人多角色模拟 |
| I | 运气族 | 随机、抽奖、悬念 | ★★★★★ 天然适配 |
| J | 韵律族 | 歌谣、拍手、节奏 | ★★ 身体参与难转化 |
| K | 笔纸族 | 连线、填空、涂色 | ★★★★★ 天然适配 |
| L | 运动族 | 跑、跳、投、蹲 | ★ 身体运动难转化 |

对于 HTML 原型，优先选择评级 ★★★★ 及以上的族系。J/L 族系可做概念性改编，但需要为屏幕上操作进行大幅重新构想。

每款游戏按 `references/game_design_doc_template.md` 中的模板生成游戏设计文档。GDD 必须包含：

1. **游戏名称** —— 朗朗上口，中文，适龄
2. **游戏机制类型** —— 来自玩法机制总表，含编号（如"A01 抢答赛"、"E02 密室逃脱"）
3. **核心概念** —— 一段话的电梯演讲
4. **教学目标映射** —— 该游戏对应哪些教学步骤？
5. **核心玩法循环** —— 逐步描述玩家的操作流程
6. **思维激发** —— 该游戏激发哪种思维方式？（逻辑推理/空间想象/策略规划/创造性思维/归纳总结/……）
7. **视觉风格** —— 简要的美术方向说明
8. **难度递进设计** —— 游戏如何逐级进阶？
9. **反馈系统** —— 游戏如何告诉学生答对/答错？
10. **HTML 原型范围** —— 原型将包含哪些功能？

### 步骤三：构建 HTML 原型与嵌入式自适应练习

为每款游戏构建一个独立的、可玩的 HTML 文件。每个原型必须包含两个集成系统：**核心游戏**和**自适应练习模块**。

**技术要求：**
- 单文件 `.html` —— 无外部依赖（所有 CSS/JS 内联）
- 离线可用 —— 不使用 CDN，不发起网络请求
- 响应式 —— 适配桌面浏览器和平板设备（最小宽度：320px）
- 触控友好 —— 按钮/可点击区域至少 44×44px
- 中文界面
- 简洁、适合儿童的视觉设计

**游戏玩法要求：**
- 清晰的胜负条件
- 每个操作即时反馈
- 分数或进度追踪
- "重新开始"按钮
- 至少 3 个关卡/轮次的内容
- 游戏内难度递进

**自适应练习要求（内嵌于游戏中）：**
- 游戏通关后，胜利界面显示"进入练习"按钮
- 练习难度根据游戏正确率**自动选择**：
  - < 40% 正确率 → 第一档（基础巩固）
  - 40-60% 正确率 → 第二档（查漏补缺）
  - 60-80% 正确率 → 第三档（能力提升）
  - 80-95% 正确率 → 第四档（思维拓展）
  - >= 95% 正确率 → 第五档（挑战冲刺）
- 每档包含 3-5 道题目，题目来源于按 `references/practice_problems_template.md` 定义的五档题目池
- 练习模式每道题即时显示正确/错误反馈，并附解题提示
- 练习完成后显示总结界面，展示游戏 + 练习的综合结果
- "直接重玩"跳过选项始终可用，可跳过练习

**代码质量：**
- 注释清晰的 JavaScript
- 游戏逻辑、练习逻辑和渲染逻辑清晰分离
- 在游戏过程中追踪 `correctCount` 和 `totalAttempts`，用于计算练习档位
- 处理边界情况（快速连击、浏览器缩放、空输入等）
- 包含带有游戏名称的 `<title>` 标签

使用 `assets/game_template.html` 作为起始脚手架。该模板已包含：
- 完整的练习模式界面（默认隐藏）
- `determinePracticeTier()` 自适应档位选择逻辑
- `enterPracticeMode()` / `renderPracticeQuestion()` / `showPracticeSummary()` 完整流程
- 选择题和填空题两种题型，均含反馈

**如何针对每款游戏自定义：**
1. 将游戏逻辑部分替换为你的游戏机制代码
2. 将 `practiceProblems` 对象替换为 5 档真实题目
3. 根据需要自定义 `difficultyLabels` 档位名称
4. 确保游戏逻辑追踪 `state.correctCount` 和 `state.totalAttempts`

### 步骤四：生成练习题目池

针对当前知识点，设计将嵌入到每个游戏原型中的 5 档练习题目池。遵循 `references/practice_problems_template.md`。

**关键设计规则：**
1. 每档 3-5 道题，难度匹配档位
2. 第 1-2 档：直接应用，建立信心。第 3 档：多步骤，加深理解。
   第 4 档：新情境，知识迁移。第 5 档：开放性，创造性思维。
3. 题型混合：整个题目池中至少包含一道填空题和一道开放性/推理题
4. 每道题必须包含 `hint` —— 简短的解题思路，教会方法而不只是给出答案
5. 同一题目池在所有 5 个游戏原型中复用（各游戏变化的只是玩法，而非练习内容）

**写入每个游戏原型：**
将 `practiceProblems` 对象复制到每个 HTML 文件的 `<script>` 部分，替换模板中的示例数据。练习界面和自适应逻辑已在模板中预先构建。

## 游戏设计原则

每一款游戏设计都应遵循以下原则：

### 1. 先思考，再点击
游戏应要求脑力投入。避免设计成学生可以靠随机点击就能通关的形式。每一次交互都应是一个决策。

### 2. 犯错即学习
错误答案应具有信息价值，而非惩罚性。展示答案为什么是错的。允许重试。绝不使用负面语言或羞辱。

### 3. 展示数学
让数学思维可视化。使用数轴、面积模型、操作工具和视觉表征。不要让华丽的特效掩盖数学本身。

### 4. 渐进的挑战
从直接应用开始，建立信心；然后引入需要更深入理解的变式。最后一关应要求学生将知识迁移到新情境中。

### 5. 赞美思考，而不只是答案
奖励策略、进步和创造性方法——而不只是正确答案。加入"思维加分"或对巧妙解法的认可。

### 6. 自适应练习闭环
游戏玩法是钩子——练习才是真正的学习。内嵌的自适应练习系统确保每位学生都能获得适合自己水平的题目：基础薄弱的学生重建根本，能力强的学生进一步拓展。这避免了静态练习册"一刀切"的陷阱。

### 7. 中国数学教育语境
- 对齐人教版课程标准
- 使用中文数字格式和习惯
- 如适用，遵循中国股市颜色惯例（红涨绿跌）
- 所有数字形式均支持中文字符显示
